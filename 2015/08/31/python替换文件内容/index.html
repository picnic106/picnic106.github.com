需求

这几天在把博客从octopress迁移到hexo，因为octopress是用ruby写的，依赖了太多东西，而且生成文章慢。迁移过程中我发现三个不同点：

  1. octopress中的文章yaml标记和hexo还是有一些区别，例如octopress中的categories更像hexo中的tags，而hexo中的categories是有层次和序列的，和octopress中的categories不是一个概念。
  2. octopress中使用的代码高亮插件是coderay，但是在hexo中却是codeblock。
  3. 在hexo中，代表段落的#，后面需要加空格，但是octopress却不需要。
  4. octopress中如果要加目录需要使用,但是在hexo中，不需要这么做，hexo会自动生成:

所以这里有三个地方需要批量操作文件修改，修改yaml、将coderay替换为codeblock、删除octopress中的目录生成代码。
其实一直想学python，正好有机会，就拿这个批量修改文件的需求练练手。

python批量修改文件

通过一天的学习发现python相对于java有几大不同点：

  1. 变量不用申明什么类型，java是一定要申明的;
  2. 不用将代码内容放到大括号内，python使用四个空格缩进来表明一个方法或者一个类的结束，for或者条件判断使用冒号，显示进入代码内容;
  3. 最大的不同python是脚本语言，不用编译再运行，直接运行即可；
  4. 代码相较于java要简洁许多，例如打开文件流，java至少四五行，而python一行就行:open(fileObj,'r+',encoding= 'utf-8')。
